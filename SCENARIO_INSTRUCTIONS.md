# Scenario Instructions

This file is now a lightweight entry point.
Use the canonical template below for all future user-flow requests:

- `docs/USER_FLOW_TEST_REQUEST.md`

Why this change:
- Single source of truth for flow requirements
- Consistent positive/negative coverage definition
- Standard execution and evidence checklist
- Easier long-term maintenance as framework grows

How to use:
1. Open `docs/USER_FLOW_TEST_REQUEST.md`.
2. Fill all required sections for your new flow.
3. Share the completed file, and tests will be generated in `tests/flows/<domain>/`.
